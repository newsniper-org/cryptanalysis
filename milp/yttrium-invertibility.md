# yttrium 가역성 — 덧셈형 Horst 조건 전수 검증 (결정적 음성)

> 도구: `milp/yttrium_invert_check.cu` (GPU 비트맵, 2³² 전수 = **완전 증명**).
> 결론: **현행 ARX 결합기(전-레인 broadcast)는 순열이 아니다.**
>
> **▶ 후속 해결(중요):** 아래 §4의 갈래 A(Feistel)·B(비가역 모드교체)는 **모두 폐기**됐다.
> 제3의 길 — **영합(zero-sum) Lai-Massey reduction** — 이 덧셈형 Horst 치환성 요구 자체를
> 우회한다(`Σεᵢ(x'ᵢ⊞t)=S`, Σε=0). F·결합기 비가역과 무관하게 라운드 전단사. Feistel 불요,
> 비가역 모드교체 불요, Farfalle 유지. 실측·확정: **`milp/yttrium-lm-results.md`**,
> `milp/yttrium_lm_invert.py`(2²⁴ 전수 전단사 ✓). 본 문서는 *왜 단순 가산 broadcast가
> 비가역인지*의 기록으로만 유효하다.

## 1. 가역 조건의 유도

라운드 = `ι → ARX 결합기 → σ → π`. ι(XOR 상수)·σ(GF α-곱, bijection)·π(워드 치환)는
모두 자명한 치환이므로 **라운드 가역 ⟺ 결합기 가역**.

결합기: `yᵢ = ROTR_β(ROTL_α(xᵢ) ⊞ t)`, `t = F(S)`. 회전을 흡수해 `x'ᵢ=ROTL_α(xᵢ)`,
`y'ᵢ=ROTL_β(yᵢ)=x'ᵢ ⊞ t`로 두면, **모듈러-합 reduction** `Σ=⊞ᵢ x'ᵢ` 채택 시:

```
⊞ᵢ y'ᵢ = (⊞ᵢ x'ᵢ) ⊞ 8t = Σ ⊞ 8·F(Σ) =: Φ(Σ)
```

복원은 출력합 `⊞y'ᵢ`로부터 Σ를 되찾는 것에 달려 있으므로:

> **가역 ⟺ Φ(Σ) = Σ ⊞ 8·F(Σ) 가 Z/2³²의 치환** ← *덧셈형 Horst 조건*

(현행 SPEC의 XOR-reduction `S=⊕xᵢ`은 더 나쁘다: `⊟t`가 ⊕에 분배되지 않아 t에 대한
*고정점 방정식* `t=F(ROTR_α(⊕ᵢ(y'ᵢ⊟t)))`이 되며 유일해 보장이 없다. 깔끔한 Horst
형태조차 모듈러-합 reduction에서만 나온다.)

## 2. 전수 결과 (2³², GPU 비트맵)

`Φ` 치환 ⟺ surjective ⟺ distinct 이미지 = 2³². 512MB 비트맵으로 이미지 크기 측정:

| c (in `Σ ⊞ c·F(Σ)`) | distinct / 2³² | 도달률 | 판정 |
|---|---|---|---|
| **8** (설계값, 8t) | 2 640 514 472 | **61.48%** | ✗ NOT bijective |
| 2 | 2 698 236 456 | 62.82% | ✗ |
| 1 (홀수 단위원) | 2 221 831 527 | 51.73% | ✗ |
| 16 | 2 709 583 725 | 63.09% | ✗ |
| 24 | 2 670 857 620 | 62.19% | ✗ |

**모든 c에서 비치환.** 가산계수 튜닝으로 고칠 수 없음(c=1 홀수에서도 실패).

## 3. 원인 — 본질적, 수정 불가(현 구조 내)

`8·F = F≪3`. 출력 비트 j = `Σ_j ⊞ (F≪3)_j`, 여기서 `(F≪3)_j = F_{j-3}`.
T-함수(삼각) 가역성의 충분조건은 "`F_{j-3}`가 Σ의 **하위** 비트(<j)에만 의존"인데,
`F`는 **회전** 기반이라 `F_k = Σ_k ⊕ (Σ_{k-7 mod 32} ∧ Σ_{k-17 mod 32}) ⊕ …` 로
작은 k에서 **상위 비트로 wrap** 한다 → T-함수 아님 → `Σ↦Σ⊞F(Σ)` 비치환.

즉 비가역성은 **회전 기반 F + 가산 broadcast** 구조 자체의 성질이며, c·F의 c, σ-레인,
(α,β) 어느 것으로도 복구되지 않는다. (σ-GLM이 가역이었던 건 XOR-broadcast에서 짝수
개의 t가 ⊕로 상쇄됐기 때문 — 가산엔 그 상쇄가 없다.)

## 4. 결론과 갈래 (→ SPEC §2.1/§10-A 확정)

덧셈형 Horst 가역은 **불가**(증명 완료). 둘 중 하나로 확정해야 한다.

- **A. 가역 유지 (Farfalle 보존):** 전(全)-레인 broadcast를 **일반화 Feistel(Horst)**
  로 재구조화. tap-절반이 `t` 생성, target-절반만 `Bᵢ ⊞= t`, 이후 swap. → F 조건 없이
  **구성상 가역**, ARX·no-S-box 유지, Farfalle 모드 대부분 보존. 비용: 절반만 섞이므로
  완전 확산에 **~2배 라운드**.
- **B. 비가역 압축 모델 (현 §2 broadcast 유지):** 전-레인 ARX broadcast(최강 확산)를
  그대로 두고 **비가역 수용**. 해시엔 가역 불필요, preimage 저항엔 오히려 유리. 단
  **Farfalle는 순열을 요구**하므로 모드를 **압축함수 트리(Merkle-tree) + Davies–Meyer
  feed-forward**로 교체해야 함. 비용: 새 모드 + 새 모드 보안 분석.

해시(B+tree 디렉터리 엔트리 해시)라는 용도상 B가 자연스럽고 §2.1의 저자 주석과도 정합.
A는 기존 Farfalle 인프라 변경을 최소화. **모드/구조 결정이라 저자 확정 사항.**
